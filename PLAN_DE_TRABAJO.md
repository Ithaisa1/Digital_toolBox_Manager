# Plan de trabajo

## Sprint de 7 días

### Día 1
- Definir la idea del producto y la estructura del proyecto.
- Montar backend, frontend y base de datos.
- Crear el esquema inicial de Prisma y la base de rutas.

### Día 2
- Implementar autenticación con registro, login y perfil.
- Proteger rutas con JWT.
- Preparar navegación principal y estado de usuario.

### Día 3
- Construir el CRUD de herramientas.
- Añadir categorías, estados y validaciones.
- Conectar el frontend con la API.

### Día 4
- Implementar suscripciones y renovaciones.
- Añadir alertas, movimientos y lógica de historial.
- Empezar el detalle visual de cada herramienta.

### Día 5
- Diseñar el dashboard con métricas.
- Aplicar tema claro/oscuro e internacionalización.
- Pulir la UI con un estilo minimalista azul, blanco y negro.

### Día 6
- Revisar errores, refactorizar componentes y corregir bugs.
- Añadir pruebas de salud y utilidades críticas.
- Verificar build del frontend y ejecución del backend.

### Día 7
- Preparar despliegue de backend y frontend.
- Documentar variables de entorno, migraciones y comandos.
- Hacer una revisión final antes de entregar.

## Resultado esperado
- Una aplicación funcional full stack.
- Interfaz limpia y coherente.
- Base técnica lista para producción y mantenimiento.
